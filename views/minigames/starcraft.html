Marine Vs. Zerglings

Help the marine survive the swarm.

GoGoGo!

Objective:

The swarm is coming! Zerglings will charge at you. survive for 1 minute. Their speed will increase every 30 seconds. Keep an eye on your ammo.

Controls:

  • Use Left and Right Arrow keys to move
  • Press "A" to fire your gun
  • Press "R" to reload

Unfortunately, this game currently does not support mobile controls

Story Behind the Game

This game was the final project for a Java course back in high school. The game was originally made with Greenfoot.

I made this game in three days. I was a real rookie in making games (and coding) at that time, so I just thought of the easiest thing to make, and tried to make it somewhat fun to play. I was very much addicted to Starcraft and Carbot Animation at the time, so that was what I went for.

Making the game was quite a bit harder than I thought. I went for the easiest game to make in terms of coding, I only had to code the movements, random spawning, and simple collision detections (simply detect with object positions). But I still had to code the animation frame by frame. I had to screen shot each frame and cut them out with photoshop (sorry Carbot).

The earliest version of the game had an issue of losing the game even though zerglings did not touch the marine, which was caused by me not properly resizing the image after cutting out the white spaces, causing error in collision detection. Also, the player had to hold down the "r" key to reload because otherwise, the animation would not play properly (I did not know how to code call back functions at that time).

A surprise for me was that I won the "first prize" by popular voting with this game in my class. It was not really a competition, but it was still a rewarding experience.

Recently, I remade this game with Cocos Creator when I was learning about the engine, and this is the version you are seeing. The game is definitely still buggy, I only went so far as to recreate what I had, and have not yet spend any time optimizing it. I probably should, but that will likely be done in the future.
